Plan one page only.



Ask for or infer the page name.

Then:

1\. Read project memory files.

2\. Create a page blueprint for that page.

3\. Define:

&#x20;  - page goal

&#x20;  - target audience

&#x20;  - primary CTA

&#x20;  - SEO intent

&#x20;  - required sections

&#x20;  - required proof

&#x20;  - factual dependencies

&#x20;  - internal links

&#x20;  - open questions

4\. If the project already contains a matching page or large existing file:

&#x20;  - identify the relevant current file

&#x20;  - map the section structure first

&#x20;  - determine what should be kept, refactored, replaced, or removed

5\. Save it under docs/page-blueprints/\[page-name].md

6\. Update docs/page-status.md

7\. Return the final blueprint and note whether the page is ready to build.
